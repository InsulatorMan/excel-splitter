# Rust Excel 拆分工具 - 编译指南

## 项目结构

```
rust_excel_splitter/
├── Cargo.toml          # Rust 项目配置
├── build.rs            # 构建脚本
├── README.md           # 使用说明
├── src/
│   ├── main.rs         # 主程序入口
│   ├── format.rs       # 格式处理模块
│   └── lib.rs          # 库模块（可选）
```

## 编译步骤

### 1. 安装 Rust

**Windows:**
1. 下载 https://win.rustup.rs/x86_64
2. 运行 `rustup-init.exe`
3. 选择默认安装（选项 1）
4. 等待安装完成
5. 重启命令行窗口

**验证安装:**
```bash
rustc --version
cargo --version
```

### 2. 进入项目目录

```bash
cd rust_excel_splitter
```

### 3. 编译调试版本（开发测试）

```bash
cargo build
```

编译后的文件：`target/debug/excel-splitter.exe`

### 4. 编译发布版本（生产使用）

```bash
cargo build --release
```

编译后的文件：`target/release/excel-splitter.exe`

**发布版本特点：**
- 优化级别最高
- 移除调试信息
- 体积更小
- 运行更快

### 5. 运行测试

```bash
# 运行单元测试
cargo test

# 运行程序
cargo run -- --help

# 运行并传递参数
cargo run --release -- -i test.xlsx -s "Sheet1" -c "机构"
```

## 编译优化

### 优化配置（已在 Cargo.toml 中配置）

```toml
[profile.release]
opt-level = 3        # 最高优化级别
lto = true           # 链接时优化
codegen-units = 1    # 单代码生成单元
panic = "abort"      # 简化 panic 处理
strip = true         # 移除符号表
```

### 进一步优化体积

#### 使用 UPX 压缩

```bash
# 下载 UPX
# https://github.com/upx/upx/releases

# 压缩可执行文件
upx --best target/release/excel-splitter.exe
```

**效果：**
- 原始大小：约 5 MB
- UPX 压缩后：约 2-3 MB
- 启动时会自动解压

#### 使用静态链接

```bash
# 静态链接 C 运行时
RUSTFLAGS='-C target-feature=+crt-static' cargo build --release --target x86_64-pc-windows-gnu
```

## 跨平台编译

### Windows → Linux

```bash
# 安装交叉编译工具
rustup target add x86_64-unknown-linux-gnu

# 编译
 cargo build --release --target x86_64-unknown-linux-gnu
```

### Windows → macOS

```bash
# 安装交叉编译工具
rustup target add x86_64-apple-darwin

# 编译（需要 macOS SDK）
cargo build --release --target x86_64-apple-darwin
```

## 常见问题

### Q: 编译失败，提示找不到链接器

**A:** 安装 Visual Studio Build Tools 或 GCC：

```bash
# 选项 1：安装 Visual Studio Build Tools
# https://visualstudio.microsoft.com/visual-cpp-build-tools/

# 选项 2：使用 MinGW
# 下载并安装 MinGW-w64
```

### Q: 编译后的文件还是很大

**A:** 使用 strip 和 UPX：

```bash
# 移除符号表
strip target/release/excel-splitter.exe

# UPX 压缩
upx --best target/release/excel-splitter.exe
```

### Q: 运行时提示缺少 DLL

**A:** 使用静态链接：

```bash
# 编辑 .cargo/config.toml
target.x86_64-pc-windows-msvc:
  rustflags = ["-C", "target-feature=+crt-static"]
```

### Q: 编译速度慢

**A:** 使用增量编译：

```bash
# 开发时使用
cargo build

# 发布时使用
cargo build --release
```

## 性能对比

| 编译方式 | 文件大小 | 启动速度 | 运行速度 |
|----------|----------|----------|----------|
| Debug | 10-15 MB | 慢 | 慢 |
| Release | 3-5 MB | 快 | 快 |
| Release + Strip | 2-4 MB | 快 | 快 |
| Release + UPX | 1-2 MB | 稍慢 | 快 |
